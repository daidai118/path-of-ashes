# Goal: 灰烬之路 - 技能系统详细设计

## 任务目标
填充 `/root/ai/token/game-design/systems/skills.md`，完成完整的技能系统设计。

## 背景信息
魂系像素风动作RPG。核心卖点是"技能双用系统"——每个技能同时服务战斗与探索。

技能来源三层：
1. **装备层** - 武器/护符提供基础技能
2. **升级层** - 投入点数强化，3条分支（战斗/探索/特殊）
3. **组合层** - 玩家自由搭配3主动+3被动，特定组合产生隐藏效果

## 需要完成的内容

### 1. 完整技能列表 (主动技能)

每个技能需要包含：

```markdown
#### [技能名称]

**来源:** [哪个区域/装备获得]
**消耗:** [专注值消耗量]
**冷却:** [秒数或无]

**战斗效果:**
- [详细描述]

**探索效果:**
- [详细描述]

**升级分支:**
- 战斗线：[具体效果]
- 探索线：[具体效果]
- 特殊线：[具体效果]

**视觉效果:** [动画/特效描述]
```

需要设计的主动技能（至少12个）：
- 3个基础技能（初始获得）
- 6个区域技能（每个区域一个）
- 3个高级/隐藏技能

### 2. 完整技能列表 (被动技能)

至少设计8个被动技能：
```markdown
#### [被动名称]

**来源:** [装备/升级]
**触发条件:** [什么情况下触发]
**效果:** [具体效果]
**冷却:** [如有]
```

### 3. 隐藏组合效果 (至少8组)

```markdown
| 技能A | 技能B | 组合名称 | 效果 | 发现难度 |
|-------|-------|---------|------|---------|
| xxx | xxx | xxx | xxx | 容易/中等/困难 |
```

### 4. 专注值系统
- 基础最大值
- 恢复速度（脱战/战斗中）
- 升级后的成长曲线

### 5. 技能槽位规则
- 3个主动槽的解锁条件
- 3个被动槽的解锁条件
- 是否可以重复装备同一技能

## 设计原则
- 每个技能必须有战斗和探索两种用途
- 升级分支改变行为方式，不是单纯数值提升
- 组合效果应该是玩家能自己发现的，不要太隐晦
- 技能之间要有协同和对抗关系

## 输出格式
直接修改 `/root/ai/token/game-design/systems/skills.md`，把所有待填充内容替换为具体设计。

## 验收标准
- [ ] 至少12个主动技能，每个都有完整的战斗/探索双用途设计
- [ ] 至少8个被动技能
- [ ] 至少8组隐藏组合效果
- [ ] 专注值数值设计合理
- [ ] 升级分支设计有深度
