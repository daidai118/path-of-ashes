## 二、技术架构

### 2.1 核心架构模式

| 模式 | 说明 | 优势 |
|------|------|------|
| 自动加载单例 | GameManager, AudioManager, EventBus, SaveManager | 全局状态管理 |
| 信号驱动 | 系统间通过EventBus通信 | 解耦、易维护 |
| 状态机 | 敌人、玩家、Boss使用状态机 | 清晰的状态管理 |
| 组合优于继承 | 技能、装备使用组合模式 | 灵活、可扩展 |

## 五、核心系统设计

### 5.1 技能系统

**设计理念：** "每个技能都是一把钥匙和一把剑"

- 每个技能同时服务战斗与探索
- 技能组合产生隐藏效果
- 技能决定你能到达的区域

**技能列表：**
| 技能 | 来源 | 战斗效果 | 探索效果 |
|------|------|---------|---------|
| 翻滚闪避 | 初始 | 无敌帧位移 | 躲避陷阱 |
| 锈蚀钩索 | 初始 | 拉近敌人 | 钩住钩环 |
| 灰烬回响 | 腐朽之母 | 残影战斗 | 残影踩机关 |
| 圣光术 | 堕落主教 | 范围伤害 | 照亮黑暗 |
| 震地击 | 矿坑蠕虫 | 范围击倒 | 破坏地形 |
| 血誓之刃 | 被放逐的国王 | 牺牲HP换伤害 | 血祭机关 |
| 暗影步 | 深渊守望者 | 瞬移闪避 | 穿过障碍 |
| 火焰印记 | 火焰守护者 | 附加火焰 | 点燃可燃物 |

### 5.2 战斗系统

**核心理念：** 魂系战斗哲学

- 有规律可循 — 每个敌人的攻击都有模式
- 高风险高回报 — 贪刀会死，耐心输出有奖励
- 死亡是学费 — 每次死亡都学到东西
- 多种解法 — 同一个敌人可以用不同策略应对

**伤害计算公式：**
```
基础伤害 = 武器基础伤害 × (1 + 力量% + 武器补正%)
技能伤害 = 基础伤害 × 技能倍率 × (1 + 信仰%)
最终伤害 = 技能伤害 × 属性克制倍率 × (1 - 防御减伤%)
```

### 5.3 区域设计

**世界地图结构：**
```
初始墓地 → [三选一] → 灰烬主城 → [三选一] → 灰烬之心
```

**9个区域：**
1. 初始墓地（教程）
2. 腐朽森林（探索）
3. 沉没教堂（战斗）
4. 灰烬矿坑（技能）
5. 灰烬主城（安全区）
6. 王城废墟（中期）
7. 深渊回廊（中期）
8. 火焰神殿（中期）
9. 灰烬之心（最终）
